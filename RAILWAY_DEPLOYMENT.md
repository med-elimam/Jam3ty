# Railway Deployment Guide

This guide explains how to deploy the Jam3ty project to Railway with PostgreSQL support.

## Prerequisites

- GitHub repository configured as the source of truth
- Railway account and project created
- PostgreSQL database provisioned on Railway

## Architecture

The deployment consists of two main services:

| Service | Purpose | Port | Environment |
|---------|---------|------|-------------|
| `api` | Express.js backend API | 3000 | Node.js |
| `postgres` | PostgreSQL database | 5432 | PostgreSQL |

## Railway Services Setup

### 1. PostgreSQL Database

1. In Railway, create a new PostgreSQL service
2. Note the `DATABASE_URL` connection string (automatically generated)
3. This will be available as an environment variable in the API service

### 2. API Service

1. Connect your GitHub repository to Railway
2. Select the repository and branch (typically `main`)
3. Railway will auto-detect the Node.js project

## Environment Variables

Configure the following environment variables in Railway:

### Required Variables

| Variable | Value | Example |
|----------|-------|---------|
| `NODE_ENV` | Set to `production` | `production` |
| `PORT` | API server port | `3000` |
| `DATABASE_URL` | PostgreSQL connection string | Auto-generated by Railway |
| `JWT_SECRET` | JWT signing secret (generate with `openssl rand -base64 32`) | Random 32-byte base64 string |
| `JWT_REFRESH_SECRET` | JWT refresh secret (generate with `openssl rand -base64 32`) | Random 32-byte base64 string |

### Optional Variables

| Variable | Purpose | Example |
|----------|---------|---------|
| `CORS_ORIGINS` | Comma-separated list of allowed origins | `https://student.jamiati.com,https://admin.jamiati.com` |
| `EXPO_PUBLIC_API_BASE_URL` | Mobile app API base URL | `https://api.jamiati.com` |

### Generate Secrets

Generate secure secrets using OpenSSL:

```bash
# Generate JWT_SECRET
openssl rand -base64 32

# Generate JWT_REFRESH_SECRET
openssl rand -base64 32
```

## Build and Start Commands

### Build Command

```bash
pnpm run build
```

This builds the API server using esbuild and generates the `dist/index.mjs` file.

### Start Command

```bash
NODE_ENV=production node --enable-source-maps ./dist/index.mjs
```

This starts the API server in production mode with source maps enabled for better error tracking.

### Migration Command

```bash
pnpm run migrate
```

This runs database migrations using Drizzle Kit. Run this before the first deployment and after schema changes.

### Force Migration Command (Use with Caution)

```bash
pnpm run migrate:force
```

This forces database migrations. Use only if you need to reset the schema. **This is destructive and should only be used in development or with explicit intent.**

## Health Check

The API includes a health check endpoint that returns the server status:

```bash
curl https://api.jamiati.com/api/healthz
```

Expected response:
```json
{
  "status": "ok",
  "timestamp": "2024-07-03T12:00:00.000Z"
}
```

## Static File Serving (Future)

The API is configured to serve static files for student web and admin dashboard in production:

- **Student Web**: Served at `/` (if `public/student/index.html` exists)
- **Admin Dashboard**: Served at `/admin` (if `public/admin/index.html` exists)
- **API Routes**: Served at `/api`

To enable this:

1. Build your student web and admin apps
2. Place the build outputs in `public/student` and `public/admin` directories
3. Redeploy the API service

## CORS Configuration

The API is configured to accept requests from specified origins. Configure `CORS_ORIGINS` as a comma-separated list:

```
https://student.jamiati.com,https://admin.jamiati.com,http://localhost:3000
```

## Mobile App Configuration

For Expo Go development and production builds, configure:

```bash
EXPO_PUBLIC_API_BASE_URL=https://api.jamiati.com
```

This environment variable is read by the mobile app to determine the API base URL.

## Troubleshooting

### Database Connection Issues

1. Verify `DATABASE_URL` is set correctly in Railway
2. Check that PostgreSQL service is running
3. Ensure the API service has access to the PostgreSQL service

### Build Failures

1. Check the build logs in Railway
2. Ensure all dependencies are installed: `pnpm install`
3. Verify TypeScript compilation: `pnpm run typecheck`

### CORS Errors

1. Verify `CORS_ORIGINS` includes the client domain
2. Check that the client is sending requests to the correct API URL
3. Test with `curl` to verify the API is responding

### Health Check Failures

1. Verify the API is running: `curl https://api.jamiati.com/api/healthz`
2. Check the API logs in Railway
3. Ensure `NODE_ENV=production` is set
