Current state:
Phase 1 Arabic language cleanup is done.
The app works, but the visual design is still weak and does not feel like a commercial product.

Goal:
Create a clean, modern, premium mobile design system for the Jamiati app.

Brand:
- Arabic app name: جامعتي
- Latin: Jamiati
- Subscription: جامعتي بلس / Jamiati Plus
- Tagline: جامعتك في جيبك

Design direction:
- Professional university student app
- Modern, clean, premium
- Not childish
- Not generic
- Not overloaded
- Arabic-first
- Mobile-first
- Suitable for Mauritanian university students
- Must feel like a real commercial product, not a hackathon prototype

Main visual improvements required:

1. Create a centralized theme system
Create or clean a shared theme file containing:
- colors
- spacing
- radius
- typography
- shadows/elevation
- card styles
- button styles
- input styles
- header styles
- badge styles

Do not hardcode random colors in every screen.

Suggested palette:
- Primary: deep academic blue
- Secondary: lighter blue
- Background: very dark navy or clean off-white depending on current theme
- Card background: slightly elevated navy card
- Accent/Plus: warm gold
- Danger: red
- Success: green
- Text primary: high contrast
- Text secondary: muted but readable
- Border: subtle blue/gray

2. Improve typography
- Use consistent font sizes.
- Fix oversized or awkward titles.
- Arabic text must be readable.
- Section titles must be strong but not huge.
- Body text must not look faded.
- Empty state text must be readable.

3. Improve headers
All major screens should have clean headers:
- الرئيسية
- المواد
- الجدول
- المجتمع
- الملف الشخصي
- المزيد
- الإعدادات
- الاشتراك

Headers should look consistent.
Avoid huge empty top spaces.
Respect iPhone safe area.

4. Improve bottom navigation
Make bottom tabs feel polished:
- Clear active state
- Consistent icon sizes
- Arabic labels
- Better spacing
- No cramped layout
- No visual clutter

Tabs:
- الرئيسية
- المواد
- الجدول
- المجتمع
- الملف الشخصي

5. Improve cards
Create reusable card styling:
- Rounded corners
- Consistent padding
- Subtle borders
- Good contrast
- Pressed state if practical

Apply to:
- Home sections
- Profile sections
- More module cards
- Subscription card
- Settings rows
- Course cards
- Empty state cards

6. Improve buttons
Create consistent button variants:
- primary
- secondary
- outline
- ghost
- danger
- plus/gold

Fix:
- disabled button appearance
- large button padding
- text alignment
- loading state if already present

7. Improve forms
Apply polished form styling to:
- login
- register
- onboarding
- settings inputs if any

Inputs should have:
- readable placeholder
- proper RTL text alignment for Arabic
- good borders
- focus state if practical
- error text style

8. Improve empty states
Current empty screens feel dead.
Create attractive empty states with:
- icon
- Arabic title
- short helpful text
- optional action button where useful

Examples:
Courses:
