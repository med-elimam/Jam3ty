You are a senior full-stack product engineer. Build a complete production-grade university student platform for Mauritania.

This is NOT an MVP, NOT a prototype, NOT a demo, and NOT a WebView app.

The product must be a real full-stack application with:

* iOS app code
* Android app code
* Web student app
* Admin dashboard
* Backend API
* Database schema
* Authentication
* Role-based permissions
* Subscription system
* Local Mauritanian payment workflow
* Activation codes
* Agent/reseller system
* Security
* Seed data
* Documentation
* Production-ready structure

The app must use React, Expo, and React Native Web for the student app. It must not be a WebView wrapper.

The target market is Mauritania only.

Business model:

* B2C first: students register directly and use the app.
* B2B-ready from day one: universities, institutes, faculties, and departments can later become official partners without rebuilding the system.
* Monetization must be based mainly on local Mauritanian payment methods, not Apple Pay, Google Pay, Apple In-App Purchases, or Google Play Billing.
* The main payment methods are:
    * Bankily manual payment
    * Masrvi / مصرفي manual payment
    * Sedad manual payment
    * Cash through local agents
    * Activation codes
    * Admin-approved payment proofs
    * Agent/reseller code sales

Do not implement fake payment APIs. Keep the architecture ready for future integrations with Bankily, Masrvi, Sedad, or a local payment aggregator, but for now implement the full manual payment and activation-code system.

    Duration: 120 days

    Features:

Manual payment workflow:

1. Student selects a plan.
2. Student sees payment instructions on web or allowed platforms.
3. Student submits:
    * payment method
    * amount
    * phone number
    * transaction reference
    * screenshot/proof upload
4. Payment becomes pending/under_review.
5. Finance Admin reviews it.
6. Finance Admin approves or rejects.
7. If approved, subscription is activated automatically.
8. Student receives notification.
9. Audit log is created.

Important iOS rule:
The architecture must support backend subscription activation and activation-code redemption. Avoid hardcoding external payment links into iOS screens in a way that could violate App Store rules. The web app can show full local payment instructions.
